# Worksheet 4 — Digital Legacy Inventory

**Source:** Lines 340-353 of `evidence/source_material.md`  
**Purpose:** Ensure digital assets are accessible and managed  
**Length:** 1-2 pages  
**Time Estimate:** 10-minute start

---

## Purpose

This worksheet helps attendees inventory their digital assets, accounts, and access information. It addresses the modern reality that much of our lives (and assets) exist in digital form, and ensures family can access what they need.

---

## Emotional Outcomes

### Before State
**Emotional State:** Blind spot
- Haven't thought about digital assets
- Don't know what accounts exist
- Passwords scattered or unknown
- Unclear who should access what

### After State
**Emotional State:** Modern confidence
- Clear picture of digital assets
- Know where passwords are stored
- Clear about access permissions
- Confidence that digital legacy is handled

---

## Worksheet Structure

### Section 1: Categories

Organize by category:

#### Email
- Email accounts (personal, work, etc.)
- Access information
- Who should have access

#### Photos/Cloud
- Photo storage (Google Photos, iCloud, etc.)
- Cloud storage (Dropbox, OneDrive, etc.)
- Important files and memories
- Access information

#### Banking
- Online banking accounts
- Investment platforms
- Cryptocurrency (if applicable)
- Payment apps (PayPal, Venmo, etc.)

#### Subscriptions
- Streaming services (Netflix, Spotify, etc.)
- Software subscriptions
- Cloud services
- Memberships

#### Social Media
- Facebook, Instagram, Twitter, LinkedIn, etc.
- Access information
- What to do with accounts (delete, memorialize, etc.)

#### Devices
- Computers, tablets, phones
- Access passwords/codes
- Where devices are located
- What's stored on each

### Section 2: Key Questions

#### "Where are passwords stored?"
**Purpose:** Identify password management system

**Options:**
- Password manager (which one?)
- Written down (where?)
- Browser saved
- Unknown/forgotten

**Guidance:** Encourage secure password management system.

#### "Who should access what?"
**Purpose:** Define access permissions

**Guidance:** 
- Not everything needs to be accessible
- Some accounts may need to be deleted
- Some may need to be preserved
- Define who gets access to what

#### "What do I want deleted vs preserved?"
**Purpose:** Clarify wishes for digital assets

**Guidance:**
- Some accounts should be deleted (privacy)
- Some should be preserved (memories, important files)
- Some should be memorialized (social media)
- Document preferences

## Usage Instructions

### For Attendees
1. Start with major categories (email, photos, banking)
2. List key accounts in each category
3. Note where passwords are stored
4. Think about who should access what
5. Consider what should be deleted vs. preserved
6. Don't worry about listing everything—this is a start

### For Facilitators
- Emphasize "10-minute start" - can begin with just major categories
- Remind them they don't need to list every account
- Encourage secure password management
- This is about awareness and planning, not perfection

---

## Common Questions

**Q: Do I need to list every single account?**
A: No, focus on important accounts (email, photos, banking, major subscriptions).

**Q: What if I don't remember all my passwords?**
A: That's okay. Note what you know, and consider setting up a password manager going forward.

**Q: Should I write down passwords?**
A: Use a secure password manager if possible. If writing down, keep it in a secure location (safe, password-protected file).

**Q: What if I want some accounts deleted?**
A: Document that preference. Your executor or digital executor can handle it.

**Q: What about social media?**
A: Decide if you want accounts deleted, memorialized, or preserved. Document your preference.

## Security Considerations

- Encourage secure password manager
- Avoid writing passwords in plain text
- Consider password-protected document or safe
- Share access information securely with trusted person

### Access Control
- Not everything needs to be accessible
- Some accounts may need to be deleted for privacy
- Define who gets access to what
- Consider digital executor designation
