# Output Generation Specifications (for the AI LLM)

**Source:** Lines 471-514 of `evidence/source_material.md`  
**Purpose:** Define the specifications for generating different types of seminar outputs

---

## Overview

These specifications tell an AI LLM exactly what to include when generating different types of seminar assets. Each output type has specific requirements that must be met.

---

## 11.1 When Asked to Create a Worksheet

### Must Include

1. **Title + "10-minute start" framing**
   - Clear, descriptive title
   - "10-Minute Start" or similar framing
   - Emphasizes quick start, not perfection

2. **Checklist / blanks**
   - Checkboxes for items that can be checked
   - Blanks for written responses
   - Tables if needed (e.g., Worksheet 3)
   - Clear prompts for each section

3. **One disclaimer line if legal/medical-adjacent**
   - "This is educational, not legal advice." (if legal content)
   - Must be prominent for Worksheet 6
   - Include when content touches on legal/medical topics

4. **Emotional before/after labels (if requested)**
   - Before state: what they feel entering
   - After state: what they should feel after
   - Optional but helpful for understanding outcomes

### Format Requirements

- **Length:** 1-2 pages maximum
- **Language:** Simple, plain English
- **Whitespace:** Large whitespace for easy reading/writing
- **Layout:** Clean, uncluttered, easy to scan

### Compliance Requirements

- Permission-based language
- No urgency or pressure
- Educational framing
- Self-directed completion
- Support is optional

## 11.2 When Asked to Create a One-Pager

### Must Include

1. **"Plain English" definitions**
   - Simple, clear explanations
   - Avoid jargon
   - Easy to understand
   - Educational, not sales

2. **3–5 bullets per section max**
   - Keep sections concise
   - Maximum 5 bullets per section
   - Prefer 3-4 bullets for readability
   - Each bullet should be clear and actionable

3. **"Next step" box at bottom**
   - Clear next step guidance
   - Actionable but not prescriptive
   - Permission-based
   - Can reference worksheets

### Format Requirements

- **Length:** 1 page (front and back if needed)
- **Layout:** Organized sections with clear headers
- **Visual:** Clean, scannable, easy to read
- **Tone:** Friendly, approachable, educational

### Compliance Requirements

- Educational only
- No sales language
- Permission-based
- No urgency
- Can include contact info only in One-Pager C (Support Spectrum)

## 11.3 When Asked to Create Facilitator Script

### Must Include

1. **Exact "no sales" disclaimer near beginning**
   - "Today is education only—no sales pitch."
   - Must appear early (within first 2-3 minutes)
   - Reinforce throughout if needed
   - Sets expectations clearly

2. **"Choice language" (autonomy)**
   - "You can..."
   - "If you want..."
   - "Your choice..."
   - "Optional..."
   - Reinforces autonomy throughout

3. **Q&A transition line that reduces suspicion:**
   - "No follow-up requirement from today..."
   - Reduces suspicion about hidden pitch
   - Reinforces no obligation
   - Can add: "If you want help later, we're available—only if you ask."

### Format Requirements

- **Structure:** Follow module sequence
- **Timing:** Include time estimates for each section
- **Transitions:** Clear transitions between modules
- **Language:** Conversational, not scripted-sounding
- **Tone:** Calm, factual, autonomy-respecting

### Compliance Requirements

- "No sales" disclaimer early and often
- Permission-based language throughout
- No urgency or pressure
- Educational framing maintained
- Autonomy respected

### Additional Elements

- Opening: Set expectations (education only, worksheets, optional help)
- Module introductions: Brief context for each module
- Module transitions: Smooth transitions between topics
- Closing: Canonical CTA, recap promises delivered
- Q&A handling: Permission-based, no pressure

## 11.4 When Asked to Create Follow-up Messaging

### Must

1. **Stay educational**
   - Continue educational framing
   - No sales language
   - No urgency or pressure
   - Maintain "no sales" posture

2. **Offer worksheet links/resources**
   - Provide access to worksheets
   - Share educational resources
   - Links to helpful information
   - Tools they can use independently

3. **Optional: "reply if you want help" (no booking link in body if you want strict compliance)**
   - Permission-based offer
   - No booking link in main body (for strict compliance)
   - Can include contact info if framed as optional
   - "Reply if you want help" is permission-based

### Format Requirements

- **Length:** Brief, scannable
- **Tone:** Friendly, helpful, educational
- **Structure:** Clear sections (greeting, resources, optional help)
- **Language:** Permission-based throughout

### Compliance Requirements

- Educational only
- No sales language
- No urgency
- Permission-based help offer
- No booking links (for strict compliance)
- Can include contact info if optional

### Strict Compliance Option

**If strict compliance is required:**
- No booking links in email body
- No scheduling CTAs
- Contact info only if explicitly requested
- "Reply if you want help" is the only CTA

**If moderate compliance is acceptable:**
- Can include contact info
- Can mention "if you want to schedule..."
- Still permission-based
- Still no urgency or pressure

## Integration

### With Compliance

- All outputs must pass compliance validation
- Use language bank for compliance-safe copy
- Follow guardrails (no legal advice, no overclaiming)
- Maintain "no sales" posture

### With Emotional Arc

- Outputs should support emotional arc progression
- Use preferred emotional triggers
- Avoid forbidden triggers
- Support before/after states

### With Modules

- Outputs should align with module content
- Reference module concepts appropriately
- Support module learning objectives
- Maintain educational framing

---

## Notes

- These specifications are requirements, not suggestions
- All outputs must pass compliance validation
- Quality is more important than speed
- When in doubt, err on the side of compliance
- Education and trust come first
