## SRD_DSKB Framework Integration

### System Doctrine

**Type:** Human-governed crisis-management doctrine  
**Source:** [`plans/SRD_DSKB.yaml`](plans/SRD_DSKB.yaml)

**Doctrine Principles:**
- `worst_case_ready: true` - Prepare for maximum resistance
- `roadmap_to_appellate_review: true` - All paths valid to highest court
- `human_decision_authority: true` - Humans choose, system provides options
- `purpose_is_battlefield_mapping_not_autonomous_execution: true` - Maps paths, doesn't execute

**Scope:**
- **In-Scope:** Child likeness misuse, synthetic media abuse, biometric identity misuse, school context harassment, platform removal, insurer alignment, civil/regulatory/constitutional escalation readiness
- **Out-of-Scope:** Entrapment, retaliation/vigilantism, fabrication of facts/evidence, threats of criminal prosecution as leverage

### Logic Gates System

**Location:** [`data/logic_gates.yaml`](data/logic_gates.yaml)  
**Purpose:** Scenario mapping for complete template pre-generation

**Gates:**
- **LG_01A:** Explicit minor sexual depiction
- **LG_01B:** Sexualized non-explicit
- **LG_01C:** Age ambiguous
- **LG_02:** Recognizable likeness
- **LG_03:** Economic expectancy
- **LG_04:** Education context
- **LG_05:** Platform notice
- **LG_06:** Biometric processing

**Usage:** Pre-generate templates for all logic gate combinations. Humans assess situations against gates and select appropriate pre-generated artifacts.

**Cross-References:**
- Logic Gates → Evidence Classification (language strength)
- Logic Gates → Sexualization Levels (drafting posture)
- Logic Gates → Escalation Tiers (pathway activation)

### Evidence Classification

**Location:** [`data/evidence_classification.yaml`](data/evidence_classification.yaml)  
**Purpose:** Classify evidence strength for appropriate language variants

**Classes:**
- **E1:** Direct evidence (strongest language)
- **E2:** Platform confirmations (strong language)
- **E3:** Derived similarity (moderate language, no actor ID)
- **E4:** Testimonial (moderate language)
- **E5:** Contextual (weakest language)

**Usage Rules:**
- Never use E3 to identify actor without correlating evidence
- E1 or E2 required for strong attribution claims
- Weaker evidence limits asserted certainty, not preservation/removal requests

**Cross-References:**
- Evidence Classes → Drafting Rules (language strength)
- Evidence Classes → Template Generation (variant selection)

### Citation Modes

**Location:** [`docs/legal/citation_modes.md`](docs/legal/citation_modes.md)  
**Purpose:** Jurisdiction-agnostic (Mode A) vs. jurisdiction-specific (Mode B) templates

**Mode A (Agnostic):**
- Category-level references only
- No statute numbers
- Universal applicability

**Mode B (Specific):**
- Statute numbers included
- Requires jurisdiction overlay
- State-specific rules

**Overlay System:** → [`callables/jurisdiction_overlays/`](callables/jurisdiction_overlays/) (to be created)

**Cross-References:**
- Citation Modes → Template Pre-Generation (variant requirements)
- Citation Modes → Drafting Rules (citation requirements)
